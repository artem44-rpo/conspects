Winget — это официальный менеджер пакетов для Windows, разработанный Microsoft, который упрощает установку, обновление и управление приложениями через командную строку. Он работает на Windows 10 (версия 1809+) и Windows 11, интегрируется с App Installer и использует централизованный репозиторий пакетов (WinGet Community Repository). Это аналог apt для Linux или brew для macOS, позволяющий автоматизировать развертывание софта без браузера.

## Основные возможности

- Поиск и установка тысяч программ из официального источника.
- Автоматическое обновление приложений в фоне.
- Экспорт/импорт списков установленных пакетов для быстрого клонирования окружения.
- Поддержка нескольких источников пакетов (Microsoft, сообщество).
- Синтаксис: `winget <команда> <идентификатор>` (идентификатор — имя пакета, ID или ключевое слово).


## Ключевые команды

Вот основные команды для повседневного использования. Запускайте в PowerShell или CMD от имени администратора для полной функциональности.

|Команда|Описание|Пример|
|---|---|---|
|`winget search <запрос>`|Поиск пакетов по имени или ключевому слову. Показывает ID, версию, издателя.|`winget search vscode` — найдет Visual Studio Code.|
|`winget install <ID>`|Установка пакета (интерактивно или silently). Добавьте `--silent` для безоконного режима.|`winget install Google.Chrome` или `winget install 7zip.7zip`.|
|`winget show <ID>`|Детальная информация о пакете (описание, версии, ссылки).|`winget show Python.Python.3".|
|`winget upgrade`|Показать доступные обновления.|`winget upgrade --all` — обновить все программы.|
|`winget uninstall <ID>`|Удаление пакета.|`winget uninstall Mozilla.Firefox`.|
|`winget list`|Список всех установленных пакетов через winget.|`winget list --source winget` — только из репозитория.|
|`winget export [-o файл.json]`|Экспорт списка установленных пакетов в JSON.|`winget export -o myapps.json`.|
|`winget import файл.json`|Импорт и установка пакетов из JSON.|`winget import myapps.json --silent`.|
|`winget source`|Управление источниками (обновление, добавление).|`winget source update` — обновить индекс пакетов.|
|`winget settings`|Открыть настройки (UI для кастомизации).|-|
|`winget features`|Проверка доступных функций.|-|
|`winget hashfile файл.exe`|Генерация хэша для создания своих пакетов.|-|

## Полезные флаги

- `--id` — точный ID пакета (избегает неоднозначностей).
- `--version X.Y.Z` — конкретная версия.
- `--silent /s` — без диалогов.
- `--force` — принудительная операция.
- `--accept-package-agreements` — авто-принятие лицензий.
- `--scope machine` — установка для всех пользователей (по умолчанию user).

## Примеры сценариев для лекции

1. **Быстрая установка стека разработчика**:
    
    text
    
    `winget install Microsoft.VisualStudioCode Python.Python.3 Git.Git Node.js`
    
2. **Автоматизация новой машины**:
    
    text
    
    `winget import starter-pack.json --silent`
    
3. **Массовое обновление**:
    
    text
    
    `winget upgrade --all --silent`
    

Winget экономит часы ручной установки, идеален для IT-админов и DevOps. Для полного списка: `winget --help` или документация Microsoft.
