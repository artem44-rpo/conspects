# Конспект по протоколу удалённого подключения RFB

## Что такое RFB

**RFB (Remote Frame Buffer)** — это протокол удалённого доступа к рабочему столу компьютера. Он передаёт изображение экрана удалённого компьютера и принимает действия пользователя (движения мыши и нажатия клавиш). На основе этого протокола работает технология **VNC (Virtual Network Computing)**.

Основные возможности RFB:

- удалённое управление компьютером;
- просмотр рабочего стола в реальном времени;
- передача действий мыши и клавиатуры;
- работа через локальную сеть и Интернет;
- поддержка различных операционных систем.

---

# Требования для подключения

Для подключения необходимо:

- компьютер должен быть включён;
- на удалённом компьютере должен работать VNC Server;
- знать IP-адрес или имя компьютера;
- знать пароль для подключения;
- компьютеры должны находиться в одной сети или иметь доступ друг к другу через Интернет.

---

# Подключение через TightVNC Viewer

**TightVNC** — это бесплатная программа для удалённого управления компьютером по протоколу RFB.

## Шаг 1

Запустить программу **TightVNC Viewer**.

---

## Шаг 2

В поле **Remote Host** ввести:

- IP-адрес компьютера;
- либо имя компьютера.

Например:

```
192.168.1.15
```

или

```
PC-01
```

Нажать кнопку **Connect**.

---

## Шаг 3

При появлении окна авторизации ввести пароль, установленный на VNC Server.

---

## Шаг 4

После успешной авторизации откроется рабочий стол удалённого компьютера.

Теперь можно:

- управлять мышью;
- использовать клавиатуру;
- запускать программы;
- работать с файлами.

---

# Дополнительные настройки

Перед подключением можно изменить параметры:

- качество изображения;
- глубину цвета;
- режим полного экрана;
- масштабирование окна;
- уровень сжатия изображения.

Эти настройки позволяют увеличить скорость работы при медленном соединении.

---

# Преимущества RFB

- работает на Windows, Linux и macOS;
- простота настройки;
- небольшие требования к оборудованию;
- возможность удалённого администрирования;
- большое количество совместимых VNC-клиентов.

---

# Недостатки

- передача изображения может быть медленнее по сравнению с RDP;
- некоторые реализации VNC не шифруют соединение по умолчанию;
- для безопасной работы через Интернет рекомендуется использовать VPN или SSH-туннель;
- качество работы зависит от скорости сети.

---

# Вывод

Протокол **RFB** используется для удалённого управления компьютерами и лежит в основе технологии **VNC**. Для подключения можно использовать программу **TightVNC Viewer**, в которой необходимо указать IP-адрес удалённого компьютера и пароль. После подключения пользователь получает полный доступ к рабочему столу удалённого устройства.
